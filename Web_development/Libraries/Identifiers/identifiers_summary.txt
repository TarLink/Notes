SUMMARY ==>                       #

No URI:
  - more efficient (e.g. smaller) but application-specific

URL:
  - using location as identification
  - does not require resolution to retrieve location
  - but couples location with identification

Bridges:
  - publicid/FPI, OID, UUID can be URN
  - UUID can be OID

Use cases:
  - publicid/FPI: SGML
  - data URI: need full content
  - centralized:
     - human-friendly: tag
  - decentralized:
     - UUID (random)
     - NI (content checksum)
